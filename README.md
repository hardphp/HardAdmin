## 框架分层
- 1、从下往上主要分为三层：模型层model、逻辑服务层service、控制器层controller
- 2、上层可调用下层，不可下层反向调用上层，逻辑服务层可以平层调用
- 3、分层初衷是封装复用，争取重复代码不写第二遍，此类规范并无绝对标准，重在统一规范

## CRUD代码生成
- 自动生成接口端（model、service、controller、validate ）各层代码，以及前端api、列表、表单代码
